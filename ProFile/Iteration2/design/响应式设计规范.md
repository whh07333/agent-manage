## 📐 断点定义

采用业界标准断点，与 Tailwind CSS 保持一致：

```scss
// 手机: < 640px
@media (max-width: 639.98px) { ... }

// 平板: 640px - 1024px
@media (min-width: 640px) and (max-width: 1023.98px) { ... }

// PC: > 1024px
@media (min-width: 1024px) { ... }
```

## 📱 手机端适配规范 (< 640px)

### 布局规则
1. **单列布局**: 所有内容都排成单列
2. **侧边栏**: 默认隐藏，通过汉堡按钮点击弹出抽屉（从左侧滑入）
3. **栅格**: `Row` `Col` 全部变为 1 列
4. **Padding**: 左右padding从24px减为16px

### 组件适配

| 组件 | 适配方案 |
|------|----------|
| **表格** | 不使用横向滚动，改为纵向卡片列表展示，每一行一个卡片，标签+值双排展示 |
| **下拉菜单** | 改为底部全屏弹窗展示，方便大拇指点击 |
| **模态框** | 占满全屏，留出顶部关闭栏，padding 16px |
| **分页** | 每页展示条数从10条减为5条 |
| **按钮** | 最小高度 48px，确保点击区域足够 |

### 交互适配
1. **支持触摸滑动**: 标签页、卡片支持左右滑动切换
2. **点击区域**: 所有可点击元素最小尺寸 ≥ 48x48px
3. **安全区域**: 底部留出 `env(safe-area-inset-bottom)`，避免被iPhone操作条遮挡
4. **顶部**: 留出 `env(safe-area-inset-top)` 适配刘海屏

### 性能优化
1. **移动端关闭非必要动画**: 减少CPU占用，节省电量
2. **图片响应式**: 根据屏幕DPR加载不同分辨率图片
3. **按需加载**: 组件懒加载，减少初始包体积

---

## 📱 平板适配 (640px - 1024px)

### 布局规则
1. **双列布局**: 主要内容 + 侧边栏，侧边栏可折叠
2. **侧边栏**: 默认展示，可点击折叠按钮收起
3. **Padding**: 左右padding 20px

### 组件适配
- 表格保持表格展示，可轻微横向滚动（数据列不多，不需要卡片化
- 模态框居中展示，宽度 80%

---

## 💻 PC端适配 (> 1024px)

### 布局规则
1. **完整多列布局**
2. **侧边栏常驻**
3. **Padding**: 24px

---

## 🎨 视觉适配要点

### 字体大小响应式
```css
/* 手机端 */
html {
  font-size: 14px;
}

/* PC端 */
@media (min-width: 1024px) {
  html {
    font-size: 16px;
  }
}
```

### 间距响应式
- 手机端间距整体缩小约 1/3
- 保持 8px 栅格系统不变

| 桌面 | 1440px+ | 验证大屏布局 |
