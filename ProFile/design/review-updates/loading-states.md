# OpenClaw AI项目管理系统 - 数据加载状态补充说明

## 加载状态分类

### 1. 页面加载状态
**触发场景**: 页面首次加载时
**设计**: 骨架屏加载，显示页面大致布局
**交互**: 禁止所有操作，显示加载动画

### 2. 数据加载状态
**触发场景**: 数据请求时
**设计**: 骨架屏加载，显示数据区域轮廓
**交互**: 数据区域禁止操作，显示加载动画

### 3. 操作反馈状态
**触发场景**: 操作请求时
**设计**: 按钮加载状态，显示加载动画
**交互**: 按钮禁用，显示加载动画

### 4. 错误状态
**触发场景**: 请求失败时
**设计**: 错误提示，显示错误信息和重试按钮
**交互**: 允许用户重试操作

## 加载状态设计规范

### 1. 骨架屏设计
- **颜色**: 使用浅灰色 (#f8fafc) 作为骨架屏背景
- **形状**: 使用圆角矩形，模拟实际内容形状
- **动画**: 添加脉动动画，提供视觉反馈
- **持续时间**: 动画持续时间为1.5秒

### 2. 加载动画设计
- **类型**: 使用旋转加载动画
- **颜色**: 使用蓝色 (#2563eb) 作为主色调
- **大小**: 24px标准大小，根据情况调整
- **位置**: 居中显示，避免遮挡其他内容

### 3. 错误状态设计
- **颜色**: 使用红色 (#ef4444) 作为主色调
- **图标**: 使用错误图标 (🚫)
- **文字**: 显示错误信息和重试按钮
- **位置**: 居中显示，避免遮挡其他内容

### 4. 超时状态设计
- **时间**: 30秒超时
- **设计**: 显示超时提示和重试按钮
- **交互**: 允许用户重试操作
- **位置**: 居中显示，避免遮挡其他内容

## 加载状态最佳实践

### 1. 骨架屏设计
- 使用页面大致布局作为骨架屏
- 显示数据区域轮廓，避免空白
- 提供视觉反馈，告知用户正在加载

### 2. 加载动画设计
- 使用标准加载动画，避免过度设计
- 确保动画流畅，不影响页面性能
- 提供明确的加载状态指示

### 3. 错误状态设计
- 显示具体的错误信息
- 提供重试功能，允许用户重新加载
- 避免泛泛的错误提示

### 4. 超时状态设计
- 合理的超时时间（30秒）
- 显示超时提示和重试按钮
- 允许用户取消请求

### 5. 性能优化
- 限制加载动画的数量
- 优化加载动画的性能
- 避免过度使用加载动画
