# AgentManage 项目编码规范

## 1. 概述

本文档定义了 OpenClaw AI Agent项目管理系统的编码规范，旨在确保代码质量、可读性和可维护性。所有开发人员必须遵循本规范进行编码。

## 2. 通用规范

### 2.1 命名规范

#### 变量命名
- **使用有意义的名称**：变量名必须能够描述其用途
- **驼峰命名法**：使用 camelCase 命名变量
- **常量**：使用全大写字母，下划线分隔
- **布尔变量**：使用 `is`、`has`、`can`、`should` 前缀

```typescript
// ✅ 正确示例
let userName: string = '张三';
let isActive: boolean = true;
let MAX_RETRY_COUNT: number = 3;
const apiBaseUrl: string = 'https://api.example.com';

// ❌ 错误示例
let n: string = '张三';          // 名称无意义
let flag: boolean = true;       // 名称不明确
let max: number = 3;            // 不够明确
```

#### 函数命名
- **使用动词或动词短语**：描述函数执行的操作
- **驼峰命名法**：首字母小写，后续每个单词首字母大写
- **避免过长**：保持简洁，不超过 30 个字符

```typescript
// ✅ 正确示例
function getUserList(): User[] { ... }
function createProject(name: string): Project { ... }
function updateTaskStatus(taskId: string, status: string): void { ... }

// ❌ 错误示例
function getList(): User[] { ... }           // 名称不够明确
function doSomething(): void { ... }         // 名称无意义
function createProjectNameIfNotExists(name: string): Project { ... }  // 名称过长
```

#### 类命名
- **使用 PascalCase**：每个单词首字母大写
- **使用名词**：描述类的用途
- **避免缩写**：除非是广泛认可的缩写

```typescript
// ✅ 正确示例
class ProjectManager { ... }
class TaskService { ... }
class UserController { ... }

// ❌ 错误示例
class ProjMgr { ... }           // 使用缩写
class manager { ... }           // 未使用 PascalCase
```

#### 文件命名
- **使用 kebab-case**：全部小写，单词之间用连字符分隔
- **组件文件**：使用组件名作为文件名
- **工具文件**：使用 `utils` 后缀

```typescript
// ✅ 正确示例
project-list.tsx
user-service.ts
api-client.ts
task-utils.ts

// ❌ 错误示例
ProjectList.tsx      // 使用 PascalCase
projectList.tsx      // 使用 camelCase
project.list.tsx     // 使用点号
```

### 2.2 代码格式

#### 缩进和空格
- **使用 2 空格**：不要使用 Tab
- **行尾不保留空格**：删除行尾多余空格
- **运算符前后空格**：保持一致

```typescript
// ✅ 正确示例
const sum = a + b;
const isValid = a === b;
function foo(a: string, b: number) { ... }

// ❌ 错误示例
const sum = a+b;          // 缺少空格
const isValid = a===b;    // 缺少空格
function foo(a:string,b:number){ ... }  // 缺少空格
```

#### 花括号
- **使用 K&R 风格**：左花括号不换行
- **始终使用花括号**：即使只有一条语句

```typescript
// ✅ 正确示例
if (isActive) {
    doSomething();
}

if (isValid) {
    return true;
} else {
    return false;
}

// ❌ 错误示例
if (isActive)
    doSomething();

if (isValid) return true;
else return false;
```

#### 引号
- **使用单引号**：字符串使用单引号
- **JSX 属性使用双引号**：保持与 HTML 一致

```typescript
// ✅ 正确示例
const name: string = '张三';
const message: string = "Hello World";
<div className="container">...</div>

// ❌ 错误示例
const name: string = "张三";
const message: string = 'Hello World';
```

### 2.3 注释规范

#### 注释原则
- **注释率 ≥ 30%**：确保代码有足够的注释
- **解释为什么**：不要只描述是什么，要解释为什么
- **保持更新**：修改代码时同步更新注释

#### 注释类型

```typescript
// 单行注释：用于解释下一步操作
// 获取用户列表
const userList: User[] = await getUsers();

// 行内注释：用于解释复杂逻辑
const result = a + b; // 计算总和

/**
 * 多行注释：用于函数和类的文档
 * @param userId 用户ID
 * @returns 用户信息
 */
function getUserById(userId: string): User { ... }
```

#### JSDoc 注释
```typescript
/**
 * 创建项目
 * @param name 项目名称
 * @param description 项目描述
 * @param priority 优先级 (P0|P1|P2|P3)
 * @returns 创建成功的项目对象
 * @throws {Error} 项目创建失败时抛出
 */
function createProject(name: string, description: string, priority: string): Project {
    // 实现逻辑
}
```

## 3. TypeScript 规范

### 3.1 类型定义

#### 接口 vs 类型
- **使用接口定义对象结构**：更清晰，易于扩展
- **使用类型定义联合类型、交叉类型**

```typescript
// ✅ 正确示例
interface User {
    id: string;
    name: string;
    email: string;
}

type UserStatus = 'active' | 'inactive' | 'pending';
type UserWithRole = User & { role: string };

// ❌ 错误示例
type User = {
    id: string;
    name: string;
};
```

#### 类型推断
- **优先使用类型推断**：不要过度显式声明类型
- **复杂函数显式声明**：明确返回值类型

```typescript
// ✅ 正确示例
const count = 0;                    // 推断为 number
const name = '张三';                // 推断为 string
const isActive = true;              // 推断为 boolean
function getUserById(id: string) {  // 明确声明返回值类型
    return {} as User;
}

// ❌ 错误示例
const count: number = 0;            // 过度声明
const name: string = '张三';        // 过度声明
```

#### 可选属性和只读属性
- **使用可选属性表示非必需字段**
- **使用只读属性保护不变的数据**

```typescript
interface User {
    id: string;
    name: string;
    email?: string;       // 可选属性
    readonly createdAt: Date;  // 只读属性
}
```

### 3.2 函数规范

#### 函数参数
- **使用默认参数**：避免 undefined 检查
- **限制参数数量**：不超过 5 个参数
- **使用对象参数**：超过 3 个参数时使用对象

```typescript
// ✅ 正确示例
function createUser(name: string, email: string, role: string = 'user'): User {
    return { name, email, role };
}

function updateProject(config: {
    name?: string;
    description?: string;
    priority?: string;
}): void { ... }

// ❌ 错误示例
function createUser(name: string, email?: string, role?: string, status?: string): User {
    return { name, email: email || 'default', role: role || 'user' };
}
```

#### 箭头函数
- **简短函数使用箭头函数**
- **保持一致性**：整个项目统一风格

```typescript
// ✅ 正确示例
const add = (a: number, b: number): number => a + b;
const getUserList = () => userList.filter(u => u.isActive);

// 较长的函数使用普通函数
function processUserData(user: User): ProcessedUser {
    // 处理逻辑
    return {
        id: user.id,
        name: user.name.toUpperCase(),
        // ...
    };
}
```

### 3.3 模块规范

#### 导入顺序
1. 外部库导入（React, Ant Design 等）
2. 内部模块导入（项目模块）
3. 样式导入
4. 类型导入

```typescript
// ✅ 正确示例
import React, { useState, useEffect } from 'react';
import { useNavigate } from 'react-router-dom';
import { Button, Table, Modal } from 'antd';
import { ProjectService } from '@/services/project';
import { UserService } from '@/services/user';
import styles from './index.module.scss';
import { User, Project } from '@/types';
```

#### 导出方式
- **使用命名导出**：便于 tree-shaking
- **避免默认导出**：保持一致性

```typescript
// ✅ 正确示例
export const ProjectService = { ... };
export function createProject(config: ProjectConfig): Project { ... }
export interface Project { ... }

// ❌ 错误示例
export default ProjectService;
```

## 4. Sequelize 模型与数据库规范 (新增)

### 4.1 表名与引用规范

> PostgreSQL 区分大小写，Sequelize 外键引用必须与实际表名一致

#### 外键引用表名规则
- **必须使用全小写**，与 PostgreSQL 实际表名完全一致
- **禁止使用 PascalCase**

```typescript
// ✅ 正确示例
assigneeId: {
  type: DataTypes.UUID,
  allowNull: true,
  references: {
    model: "users",  // ✅ 全小写，与实际表名一致
    key: "id",
  },
}

parentId: {
  type: DataTypes.UUID,
  allowNull: true,
  references: {
    model: "tasks",  // ✅ 全小写，与实际表名一致
    key: "id",
  },
}

// ❌ 错误示例
assigneeId: {
  references: {
    model: "Users",  // ❌ 大写开头，PostgreSQL找不到表
    key: "id",
  },
}
```

### 4.2 枚举值规范

> 模型枚举值必须与数据库 CHECK 约束的值**完全一致**

#### 命名规则
- 数据库使用下划线分隔 (`in_progress`)
- 模型枚举必须**完全相同**，禁止自行改为驼峰

```typescript
// ✅ 正确示例 - 与DB CHECK一致
status: {
  type: DataTypes.STRING,
  allowNull: false,
  defaultValue: "pending",
  validate: {
    isIn: [
      [
        "pending",
        "in_progress",   // ✅ 和DB一致
        "completed",
        "blocked",
        "cancelled",
      ],
    ],
  },
}

// ❌ 错误示例 - 与DB不一致
status: {
  validate: {
    isIn: [
      [
        "pending",
        "inProgress",   // ❌ 驼峰，DB是下划线
        "completed",
        "blocked",
        "cancelled",
      ],
    ],
  },
}
```

### 4.3 模型与DDL同步规范

> 模型定义修改后，必须同步更新 `init-db.sql`

#### 强制规则
1. **添加/删除/修改字段** → 必须同步更新 `init-db.sql`
2. **修改枚举** → 必须同步更新 DB CHECK 约束
3. **新增模型** → 必须在 `init-db.sql` 添加建表语句
4. **PR 评审必须检查**：模型变更后 DDL 是否同步

### 4.4 字段映射规则

| 模型字段 (camelCase) | 数据库字段 (snake_case) | 说明 |
|---------------------|------------------------|------|
| projectId | project_id | ✅ 正确 |
| assigneeId | assignee_id | ✅ 正确 |
| parentId | parent_id | ✅ 正确 |
| dueDate | due_date | ✅ 正确 |
| deliverables | deliverables | ✅ 一致（本身就是下划线友好）|
| isArchived | is_archived | ✅ 正确 |
| createdAt | created_at | ✅ 正确 |
| updatedAt | updated_at | ✅ 正确 |

Sequelize `underscored: true` 自动转换，保持一致即可。

### 4.5 开发自测步骤

模型修改后，必须执行以下步骤验证：

```bash
# 1. 删除现有库
dropdb agentmanage

# 2. 新建库
createdb agentmanage

# 3. 执行DDL
psql agentmanage -f init-db.sql

# 4. 启动服务，观察日志
npm run start
```

**验证标准**：日志输出 `Models synchronized` 无错误 → 通过

## 5. React 规范

### 5.1 组件规范

#### 组件命名
- **使用 PascalCase**：组件名首字母大写
- **文件名与组件名一致**

```typescript
// ✅ 正确示例
// ProjectList.tsx
export const ProjectList: React.FC = () => { ... }

// ❌ 错误示例
// projectList.tsx
export const projectList = () => { ... }
```

#### 组件结构
```typescript
// ✅ 推荐组件结构
import React from 'react';
import { useState, useEffect } from 'react';
import { Button } from 'antd';
import styles from './ProjectList.module.scss';

interface ProjectListProps {
    projects: Project[];
    onSelect: (project: Project) => void;
}

export const ProjectList: React.FC<ProjectListProps> = ({ 
    projects, 
    onSelect 
}) => {
    // 状态
    const [loading, setLoading] = useState(false);
    
    // 副作用
    useEffect(() => {
        // 副作用逻辑
    }, []);
    
    // 事件处理
    const handleClick = (project: Project) => {
        onSelect(project);
    };
    
    // 渲染
    return (
        <div className={styles.container}>
            {projects.map(project => (
                <div key={project.id} onClick={() => handleClick(project)}>
                    {project.name}
                </div>
            ))}
        </div>
    );
};
```

#### Props 命名
- **使用 on 前缀**：表示事件处理函数
- **使用 handle 前缀**：表示内部事件处理函数

```typescript
// ✅ 正确示例
interface ButtonProps {
    onClick: () => void;           // 外部传入的事件处理
    onChange?: (value: string) => void;
}

const Button: React.FC<ButtonProps> = ({ onClick, onChange }) => {
    const handleClick = () => {    // 内部处理函数
        onClick();
    };
    
    return <button onClick={handleClick}>Click</button>;
};
```

### 5.2 Hooks 规范

#### Hooks 命名
- **以 use 开头**：自定义 Hook 必须以 use 开头
- **保持简洁**：每个 Hook 只做一件事

```typescript
// ✅ 正确示例
const useUserList = (params: UserQueryParams) => {
    const [users, setUsers] = useState<User[]>([]);
    const [loading, setLoading] = useState(false);
    
    useEffect(() => {
        loadUsers();
    }, [params]);
    
    return { users, loading };
};

// ❌ 错误示例
const getUserList = (params) => {  // 缺少 use 前缀
    // ...
}
```

#### 依赖数组
- **完整声明依赖**：不要遗漏依赖
- **使用 ESLint 规则**：确保依赖正确

```typescript
// ✅ 正确示例
useEffect(() => {
    const subscription = api.subscribe(id, handleChange);
    return () => subscription.unsubscribe();
}, [id, handleChange]);

// ❌ 错误示例
useEffect(() => {
    api.subscribe(id, handleChange);
}, []);  // 缺少依赖
```

### 5.3 状态管理

#### 状态放置原则
- **组件本地状态**：使用 useState
- **跨组件共享**：使用 Context 或状态管理库
- **服务器状态**：使用 React Query 或 SWR

```typescript
// ✅ 正确示例
// 本地状态
const [count, setCount] = useState(0);

// 共享状态（使用 Context）
const UserContext = createContext<UserContextType>(null);

// 服务器状态（使用 React Query）
const { data, isLoading } = useQuery(['user', userId], () => 
    fetchUser(userId)
);
```

## 7. API 规范

### 7.1 请求处理

#### 错误处理
- **统一错误处理**：使用拦截器
- **用户友好提示**：显示清晰的错误信息

```typescript
// ✅ 正确示例
try {
    const response = await api.get('/projects');

        return response.data;
    }
    throw new Error(response.msg);

    message.error('获取项目列表失败，请稍后重试');
```

- **参数校验**：使用 Zod 或 Yup

```typescript
// ✅ 正确示例
interface CreateProjectParams {

};
```

## 8. 测试规范

### 8.1 单元测试

#### 测试文件命名
- **与源文件同名**：使用 `.test.ts` 或 `.spec.ts` 后缀
- **放在源文件旁边**：便于查找

```typescript

project.service.ts
project.service.test.ts

// ❌ 错误示例
project.service.ts
__tests__/project.test.ts
```

#### 测试用例结构

```typescript
// ✅ 正确示例
describe('ProjectService', () => {
    describe('createProject', () => {
        it('应该成功创建项目', async () => {

                name: '测试项目',
```

## 11. 违规处理

### 11.1 代码质量门禁

- 代码注释率 < 30% → 警告
- 单元测试覆盖率 < 60% → 禁止合并
