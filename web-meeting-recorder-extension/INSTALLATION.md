# Web Meeting Audio Recorder & Analyzer 安装指南

## 🎯 项目概述

这是一个功能完整的 Chrome 浏览器扩展，专门用于：
- ✅ **录制网页会议音频** (Zoom, Teams, Google Meet, 飞书会议等)
- ✅ **AI智能分析音频内容** (语音转文字、说话人识别、情感分析)
- ✅ **生成结构化会议纪要** (带时间戳、说话人标注、行动项)
- ✅ **区分不同说话人** (自动识别和标注不同参会者)
- ✅ **保存到特定位置** (支持自定义保存路径和格式)

## 🚀 快速安装步骤

### 第1步：准备图标文件
```bash
# 进入项目目录
cd ~/.openclaw/workspace/web-meeting-recorder-extension

# 创建 icons 目录
mkdir -p icons

# 创建三种尺寸的图标：
# - icons/icon16.png (16x16)
# - icons/icon48.png (48x48)
# - icons/icon128.png (128x128)

# 可以使用在线工具生成图标：
# https://favicon.io/
# https://www.favicon-generator.org/
```

### 第2步：加载到 Chrome
1. 打开 **Google Chrome** 浏览器
2. 在地址栏输入: `chrome://extensions/`
3. 开启右上角的 **开发者模式**
4. 点击 **加载已解压的扩展程序**
5. 选择项目文件夹: `/Users/whh073/.openclaw/workspace/web-meeting-recorder-extension`
6. ✅ 扩展图标应该出现在工具栏 (🎤 图标)

### 第3步：配置扩展设置
1. 点击扩展图标
2. 点击 **⚙️ 打开设置**
3. 配置以下信息：

#### 🎵 音频设置
- **录音质量**: 高 (192kbps)、中 (128kbps)、低 (64kbps)
- **音频格式**: MP3、WAV、WebM
- **保存路径**: 默认 `~/Recordings/`

#### 🤖 AI 模型设置
- **语音识别**: 选择提供商 (OpenAI Whisper、Azure Speech、Google Speech)
- **说话人识别**: 开启/关闭
- **API密钥**: 输入相应的 API 密钥
- **模型名称**: 如 `whisper-1`、`latest`

#### 📄 输出设置
- **纪要格式**: Markdown、TXT、JSON、HTML
- **时间戳**: 开启/关闭
- **说话人标签**: 开启/关闭
- **自动保存**: 开启/关闭

4. 点击 **测试连接** 验证配置
5. 点击 **保存设置**

### 第4步：开始使用
1. **加入网页会议** (Zoom、Teams、Google Meet、飞书会议等)
2. **点击扩展图标** 打开控制面板
3. **点击"开始录音"** 按钮
4. **会议结束后** 点击"停止录音"
5. **点击"AI分析"** 生成会议纪要
6. **查看和下载** 分析结果

## 🔧 技术架构

### 音频录制层
- **MediaRecorder API**: 高质量音频录制
- **Web Audio API**: 实时音频处理
- **tabCapture API**: 捕获标签页音频流
- **AudioWorklet**: 低延迟音频处理

### 音频处理层
- **VAD (语音活动检测)**: 自动检测语音段
- **音频增强**: 降噪、音量标准化
- **特征提取**: MFCC、频谱特征
- **格式转换**: MP3、WAV、WebM 格式支持

### AI 分析层
- **语音识别 (ASR)**: OpenAI Whisper、Azure Speech、Google Speech
- **说话人识别**: 基于特征聚类的说话人区分
- **自然语言处理**: 摘要生成、关键词提取、情感分析
- **结构化输出**: JSON、Markdown、HTML 格式

### 数据存储层
- **本地存储**: 录音文件和元数据
- **Chrome Storage**: 设置和状态管理
- **Downloads API**: 文件下载和保存
- **IndexedDB**: 历史记录管理

## ⚙️ 高级配置

### 音频参数调优
```javascript
// 在设置中调整以下参数：
const audioConfig = {
  sampleRate: 44100,     // 采样率
  bitrate: 192000,       // 比特率 (bps)
  channels: 2,           // 声道数
  mimeType: 'audio/mp3', // 编码格式
  noiseReduction: true,  // 降噪
  autoGain: true,        // 自动增益
  echoCancellation: true // 回声消除
};
```

### AI 模型选择
```javascript
// 支持的语音识别模型：
const asrModels = {
  openai: 'whisper-1',      // OpenAI Whisper
  azure: 'zh-CN',          // Azure Speech (中文)
  google: 'zh-CN',         // Google Speech (中文)
  local: 'vosk-model-zh'   // 本地 Vosk 模型

// 说话人识别配置：
const speakerConfig = {
  minDuration: 1000,       // 最小语音段时长 (ms)
  similarityThreshold: 0.7, // 相似度阈值
  maxSpeakers: 10,         // 最大说话人数
  enableClustering: true   // 启用聚类分析
};
```

## 🐛 故障排除

### 常见问题

#### Q1: 无法获取音频权限
**解决方案**:
1. 检查 Chrome 麦克风权限设置
2. 确保会议标签页有音频输出
3. 重启 Chrome 浏览器
4. 检查系统音频设置

#### Q2: 录音质量差
**解决方案**:
1. 在设置中提高录音质量
2. 关闭浏览器其他标签页
3. 检查网络连接稳定性
4. 使用有线耳机减少干扰

#### Q3: 说话人识别不准确
**解决方案**:
1. 确保每个说话人发言清晰
2. 调整说话人识别阈值
3. 手动标注说话人姓名
4. 收集更多语音样本

#### Q4: AI 分析失败
**解决方案**:
1. 检查 API 密钥是否有效
2. 确认网络连接正常
3. 检查 API 配额和限制
4. 尝试不同的 AI 提供商
