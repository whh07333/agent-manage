## 系统性发现与改进建议

### 核心发现

> **超过一半 (55%)** 的缺陷属于 **模型/数据库不一致 + 路由未注册 + 权限缺失检查**，这是最主要的缺陷来源。

1. **根本原因**: 开发修改代码后，经常忘记同步更新 `init-db.sql`，也忘记添加必要的权限检查
2. **缺少自动化检查**: 没有自动化比对脚本，导致此类问题流入测试环境
3. **开发自测不充分**: 很多问题都是创建项目/数据库后就能发现，但开发没重建就直接推送测试

### 必须落实的改进建议

1. **必须添加 CI 自动化检查**:
   - 添加脚本 **自动比对所有模型字段** 与 `init-db.sql`
   - 检查 **所有路由注册** 确保设计路由都存在
   - 不一致/编译错误直接 **阻断合并**，不修复不能合并

2. **PR Checklist 强制检查项**:
   - 模型变更 ➜ 必须勾选「**我已同步更新 init-db.sql**」才能合并
   - API路径变更 ➜ 必须勾选「**我已更新 API 文档**」才能合并

3. **开发自测要求**:
   - 通知测试前必须 `DROP DATABASE` 重建 `CREATE DATABASE` ➜ 初始化 ➜ 验证能正常启动
   - 验证 **所有新增 API** 都能正常访问才能通知测试

4. **权限检查要求**:
   - 所有资源修改操作必须验证权限（负责人/管理员）
   - 添加单元测试验证权限规则
