## 一、测试环境要求

### 1. 基础设施
- Docker: 运行PostgreSQL数据库
- Node.js: v18+ 推荐 v22+
- npm 或 yarn: 安装依赖

### 2. 端口规划（开发/测试隔离）

| 服务 | 端口 | 说明 |
|------|------|------|
| PostgreSQL（开发） | 5433 | 开发环境数据库 |
| PostgreSQL（测试） | 5434 | 测试环境数据库 |
| 后端（开发） | 3000 | 开发后端 |
| 后端（测试） | 3001 | **测试后端** |
| 前端（开发） | 5173 | 开发前端 |
| 前端（测试） | 5174 | **测试前端** |

### 3. 数据库配置

测试数据库:
- 容器名: `agentmanage-test-postgres`
- 端口: `5434 -> 5432`
- 数据库名: `agentmanage_test`
- 用户名: `postgres`
- 密码: `postgres`

## 二、部署步骤

### 步骤1: 启动数据库容器

```bash
# 如果已经有容器，删除旧容器
docker rm -f agentmanage-test-postgres

# 启动新容器
docker run -d \
  --name agentmanage-test-postgres \
  -p 5434:5432 \
  -e POSTGRES_PASSWORD=postgres \
  -v postgres_test_data:/var/lib/postgresql/data \
  postgres:14-alpine
```

### 步骤2: 初始化数据库

```bash
# 使用init-db.sql初始化数据库表结构
cat /path/to/init-db.sql | docker exec -i agentmanage-test-postgres psql -U postgres -d agentmanage_test
```

**当前项目已经提供了完整的 `init-db.sql` 在项目根目录，包含所有表结构包括 `subscriptions`。**

### 步骤3: 编译部署后端

```bash
cd backend
cp .env.test .env
npm install
npm run build
# 后台运行
node dist/server.js > backend-test.log 2>&1 &
```

### 步骤4: 启动前端

```bash
cd ../frontend
cp .env.test .env
npm install
# 后台运行
npm run dev -- --port 5174 > frontend-test.log 2>&1 &
```

### 步骤5: 验证部署

```bash
# 等待10秒后验证健康检查
curl -s http://localhost:3001/health
# 返回 200 OK 表示部署成功
```

## 五、故障排查

### 1. 端口被占用

```bash
# 查看占用端口进程
lsof -i :3001
# 杀掉进程
kill -9 <pid>
```

### 2. 数据库连接失败

- 检查容器是否运行: `docker ps | grep postgres`
- 检查端口映射是否正确: 宿主端口 5434 映射容器端口 5432
- 检查 `.env.test` 中 `DB_PORT` 是否为 5434

### 3. 服务启动成功但是API请求超时

- 检查后端是否真的在运行: `ps aux | grep node`
- 查看日志: `tail -n 50 backend-test.log`
- 检查端口是否正确监听: `.env.test` 中 `PORT=3001`
