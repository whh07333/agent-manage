# Web Meeting Audio Recorder & Analyzer

一个功能强大的 Chrome 扩展，专门用于录制网页会议音频、保存音频文件，并调用 AI 模型进行音频分析，生成会议纪要，支持说话人区分。

## 🎯 核心功能

### 1. **音频录制**
- 录制网页会议音频（Zoom、Teams、Google Meet、飞书会议等）
- 支持多种音频格式：WAV、MP3、WebM
- 高质量音频录制（采样率、比特率可调）
- 实时音频波形显示

### 2. **音频处理**
- 自动检测会议开始/结束
- 音频分割（按时间段或说话人）
- 噪音消除和音频增强
- 音频格式转换

### 3. **AI 音频分析**
- 语音转文字（ASR）
- 说话人识别和区分
- 情感分析
- 关键词提取
- 会议内容摘要

### 4. **智能纪要生成**
- 自动生成结构化会议纪要
- 说话人标注（如：张三、李四、未知说话人1）
- 时间戳标记
- 关键决策和行动项提取

### 5. **文件管理**
- 自动保存到指定位置
- 多格式输出：TXT、JSON、Markdown、PDF
- 音频文件管理
- 搜索和整理功能

## 🔧 技术架构

### 1. **音频录制层**
- **MediaRecorder API**: 捕获网页音频流
- **Web Audio API**: 音频处理和增强
- **AudioWorklet**: 实时音频处理

### 2. **AI 分析层**
- **语音识别**: Whisper API、Google Speech-to-Text、Azure Speech
- **说话人识别**: Pyannote、SpeechBrain
- **自然语言处理**: GPT-4、Claude、本地模型
- **情感分析**: 基于文本和音频特征

### 3. **数据处理层**
- **音频分割**: VAD（语音活动检测）
- **特征提取**: MFCC、频谱特征
- **时间对齐**: 音频与文字的精确对齐

### 4. **输出层**
- **音频文件**: WAV、MP3 格式
- **文字纪要**: 结构化文本
- **可视化**: 说话人时间线、关键词云图

## 🚀 快速开始

### 安装步骤
1. **克隆项目**:
   ```bash
   git clone <repository-url>
   cd web-meeting-recorder-extension
   ```

2. **创建图标**:
   ```bash
   mkdir -p icons
   # 创建图标文件：icon16.png, icon48.png, icon128.png
   ```

3. **加载扩展**:
   - 打开 Chrome → `chrome://extensions/`
   - 开启"开发者模式"
   - 点击"加载已解压的扩展程序"
   - 选择项目文件夹

4. **配置 API**:
   - 点击扩展图标 → "设置"
   - 配置音频识别 API 密钥
   - 配置 AI 模型 API
   - 设置保存路径

### 使用流程
1. **开始会议**:
   - 加入网页会议（Zoom、Teams 等）
   - 点击扩展图标 → "开始录制"

2. **录制过程**:
   - 实时显示音频波形
   - 显示当前说话人
   - 显示录制时长

3. **结束录制**:
   - 点击"停止录制"
   - 选择分析选项
   - 等待处理完成

4. **获取结果**:
   - 下载音频文件
   - 查看会议纪要
   - 查看说话人分析报告

## 🔬 技术实现细节

### 音频录制
```javascript
// 使用 MediaRecorder API
const stream = await navigator.mediaDevices.getUserMedia({ audio: true });
const mediaRecorder = new MediaRecorder(stream, {
  mimeType: 'audio/webm;codecs=opus',
  audioBitsPerSecond: 128000
});
```

### 说话人识别
1. **特征提取**: 提取音频特征（MFCC、频谱）
2. **聚类分析**: 使用聚类算法区分说话人
3. **时间分割**: 基于VAD分割音频片段
4. **说话人标注**: 为每个片段分配说话人ID

### AI 分析流程
1. **音频预处理**: 降噪、标准化、分段
2. **语音转文字**: 使用ASR模型
3. **说话人标注**: 标注每段文字的说话人
4. **文本分析**: 摘要、关键词、情感分析
5. **结构化输出**: 生成会议纪要
