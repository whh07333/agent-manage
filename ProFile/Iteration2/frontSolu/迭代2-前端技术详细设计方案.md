## 三、模块详细设计

### 3.1 项目管理模块 - US002 修改项目配置

#### 3.1.1 功能需求
作为管理类Agent，我可以通过界面修改项目配置（周期、目标、关联Agent等），以便应对项目变更需求。

#### 3.1.2 前端设计
- **页面路径**: `/projects/:id/config`
- **布局**: 表单布局，使用 Ant Design `Form` 组件
- **表单字段**:
  | 字段 | 类型 | 验证规则 |
  |------|------|----------|
  | 项目名称 | `Input` | 必填，1-100字符 |
  | 项目描述 | `TextArea` | 可选，最多 1000 字符 |
  | 目标开始日期 | `DatePicker` | 必填 |
  | 目标结束日期 | `DatePicker` | 必填，必须晚于开始日期 |
  | 优先级 | `Select` | 必填，选项 P0/P1/P2/P3 |
  | 关联Agent | `Select` 多选 | 可选 |
  | 自定义字段 | `JsonEditor` | 可选，键值对 |

- **权限控制**:
  - 仅项目负责人或系统管理员可编辑
  - 归档项目表单禁用，只读展示
  - 无权限用户显示403提示

- **交互流程**:
  1. 进入页面加载现有项目配置
  2. 用户修改表单内容
  3. 点击保存按钮，前端验证
  4. 调用 `PUT /api/projects/{id}` API
  5. 成功提示，返回项目详情页
  6. 失败显示错误信息

#### 3.1.3 组件设计
```tsx
// 页面组件
ProjectConfig/
  index.tsx          # 主页面
  hooks.ts           # 自定义hooks
  types.ts           # 类型定义
```

#### 3.1.4 状态管理
不需要全局状态，使用页面本地状态 (useState) 即可。

### 3.2 项目管理模块 - US003 项目归档

#### 3.2.1 功能需求
作为管理类Agent，我可以归档已完成的项目，沉淀项目数据到经验库。

#### 3.2.2 前端设计
- **入口**: 在项目详情页操作下拉菜单添加"归档"按钮
- **弹窗**: 确认弹窗，要求输入归档说明（可选）
- **API调用**: `POST /api/projects/{id}/archive`
- **取消归档**: 已归档项目显示"取消归档"按钮（仅管理员可用）

- **权限控制**:
  - 仅项目负责人或管理员可归档/取消归档
  - 归档后项目详情页所有编辑按钮禁用

- **交互**:
  - 点击归档 → 确认弹窗 → 确认后调用API → 成功刷新页面 → 显示归档提示条

### 3.3 任务管理模块 - US008 任务验收

#### 3.3.1 功能需求
作为验收负责人，我可以验收任务，标记验收通过或不通过并说明原因。

#### 3.3.2 前端设计
- **入口**: 在任务详情页操作区添加"验收"按钮
- **弹窗**: 验收弹窗
  - 单选框：通过/不通过
  - 文本框：验收意见（必填）
- **API调用**: `POST /api/tasks/{id}/acceptance`
- **权限**: 仅项目负责人或指定验收人可验收

- **状态流转后UI更新**:
  - 验收通过 → 任务状态变为 "completed"
  - 验收不通过 → 任务状态变为 "in_progress"
  - 页面自动刷新状态展示

#### 3.4.1 功能需求
作为开发Agent，遇到任务阻塞时，可以提交阻塞问题，说明原因和影响范围。

#### 3.4.2 前端设计
- **入口**: 任务详情页操作区添加"上报阻塞"按钮
- **弹窗**: 阻塞上报表单
  - 阻塞原因：文本框（必填）
  - 关联任务：多选选择器（可选）

- **成功后**: 任务状态自动变为 "blocked"，页面刷新

- **阻塞解决**: 任务负责人可以点击"解决阻塞"，状态恢复为 "in_progress"

### 3.5 可视化监控模块 - US015 移动端响应式适配

#### 3.5.1 功能需求
人类用户可以在PC、平板、手机端访问，获得一致体验。

#### 3.5.2 响应式断点设计
采用 Tailwind CSS 标准断点：

| 断点范围 | 设备类型 | 布局策略 |
|----------|----------|----------|
| `< 640px` | 手机 | 单列布局，侧边栏折叠为抽屉，菜单收起 |
| `640px - 1024px` | 平板 | 双列布局，侧边栏可折叠 |
| `> 1024px` | PC | 完整多列布局，侧边栏常驻 |

#### 3.5.3 具体适配方案

**布局适配**:
- 整体栅格系统使用 Ant Design Grid `Row` + `Col`，按断点设置不同 `span`
- 侧边栏：移动端隐藏，通过汉堡按钮弹出抽屉
- 表格：移动端横向滚动，或转为卡片列表展示

**交互适配**:
- 表格分页：移动端减少每页展示数量 (10 → 5)
- 下拉菜单：移动端改为全屏底部弹窗
- 模态框：移动端占满屏幕，留出顶部关闭栏
- 支持触摸滑动切换标签页和卡片

**安全区域适配**:
- 使用 `env(safe-area-inset-bottom)` 适配iPhone底部操作条
- 使用 `env(safe-area-inset-top)` 适配刘海屏

**性能优化**:
- 移动端减少动画效果，降低CPU占用
- 图片根据屏幕密度加载不同分辨率
- 按需加载组件，减少初始包大小

#### 3.5.4 适配范围
所有现有页面 + 新增页面都需要适配：
- ✅ Dashboard 首页
- ✅ 项目列表页
- ✅ 项目详情页
- ✅ 任务列表页
- ✅ 任务详情页
- ✅ 新增的8个页面

### 3.6 数据统计模块 - US016 多维度统计分析

#### 3.6.1 功能需求
人类用户可以查看统计分析数据（Agent负载、项目效率、异常统计等）。

#### 3.6.2 前端设计
- **页面路径**: `/projects/:id/statistics`
- **布局**: 卡片式概览 + 图表展示 + 数据表格
- **组件设计**:
  - `StatsCard` - 统计卡片，展示数字标题和趋势

  - 项目级统计：项目成员可访问
  - Agent级统计：Agent本人或管理员可访问
  - 团队效率：仅管理员可访问

#### 3.6.3 可视化展示
- 任务趋势：折线图展示每日创建/完成数量
- Agent负载：柱状图展示各Agent已完成/进行中/阻塞任务
- 进度卡：展示完成率、逾期率等关键指标

### 3.7 审计合规模块 - US018 死信队列管理

#### 3.7.1 功能需求
系统管理员可以管理事件推送失败的死信队列，手动重发失败事件。

#### 3.7.2 前端设计
- **页面路径**: `/admin/subscriptions/dead-letters`
- **布局**: 分页列表 + 操作栏
- **列表字段**:
  | 字段 | 展示 |
  |------|------|
  | 事件类型 | 标签展示 |
  | 订阅ID | 链接跳转 |
  | 重试次数 | 数字展示 |
  | 最后错误 | 文本截断，悬浮查看完整 |
  | 创建时间 | 相对时间展示 |

- **操作**:
  - 单条重发：点击重发按钮，调用 `POST /api/subscriptions/dead-letters/{id}/retry`
  - 批量重发：勾选多条，点击"全部重发"，调用 `POST /api/subscriptions/dead-letters/retry-all`
  - 删除：点击删除按钮，调用 `DELETE /api/subscriptions/dead-letters/{id}`

- **分页**: 默认每页 20 条，支持切换页码

- **权限**: 仅系统管理员可访问

### 3.8 审计合规模块 - US019 API密钥全生命周期管理

#### 3.8.1 功能需求
系统管理员可以管理Agent的API密钥（颁发、轮换、吊销）。

#### 3.8.2 前端设计
- **页面路径**: `/admin/agents/:id/api-keys`
- **布局**: 密钥列表 + 新建/轮换弹窗
- **列表字段**:
  | 字段 | 展示 |
  |------|------|
  | 创建时间 | 日期 |
  | 过期时间 | 日期，过期标红 |
  | 状态 | 标签：激活/过期/吊销 |
  | 操作 | 按钮：吊销/查看 |

- **操作**:
  - 新建密钥：点击"新建密钥"，设置过期时间（默认90天），调用后端API创建
  - 轮换密钥：新建密钥后，旧密钥自动进入24小时过渡期
  - 吊销密钥：点击吊销，确认弹窗，吊销后立即失效

- **安全注意**:
  - 创建成功后只显示一次完整密钥，提示用户保存
  - 列表中只显示密钥前缀（前4位），不展示完整密钥
  - 吊销后立即从列表中标记为已吊销

- **权限**: 仅系统管理员可访问

  - 错误率超过阈值显示红色警告
  - CPU/内存使用率过高显示黄色警告

## 五、数据库与状态管理

### 5.1 状态管理设计
| 模块 | 状态管理方式 | 原因 |
|------|-------------|------|
| 项目配置修改 | 本地组件state | 表单提交后跳转，不需要全局状态 |
| 项目归档 | 调用后刷新页面 | 状态在后端，前端不需要持久化 |
| 任务验收/阻塞 | 调用后刷新页面 | 状态在后端 |
| 死信队列管理 | 本地组件state + 分页 | 列表查询，不需要全局状态 |
| API密钥管理 | 本地组件state | 管理页面，低频操作 |
| 统计分析 | 本地组件state + React Query缓存 | 缓存减少重复请求 |
| 实时监控 | 定时器刷新 | 数据需要定期更新 |

### 5.2 缓存策略
- 统计数据：使用 React Query 缓存，缓存时间 5 分钟（和后端对齐）
- 页面数据：合理设置缓存，减少不必要请求
- 实时监控：不缓存，每30秒刷新
