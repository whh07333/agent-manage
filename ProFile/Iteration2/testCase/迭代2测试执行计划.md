## 三、测试执行顺序

遵循从底层到上层的测试顺序：

```
单元测试(开发) → 接口测试 → 集成测试 → 功能测试 → 兼容性测试 → 安全性能测试 → 回归测试 → 验收测试
```

### 阶段一：接口测试（第一天）
测试所有新增API接口：
1. 参数合法性校验（空值、非法格式、越界值）
2. 权限控制验证（不同角色访问控制）
3. 成功/失败场景验证
4. 响应格式验证

### 阶段二：集成测试（第二天）
验证完整业务流程：
1. 项目配置修改流程
2. 项目归档/取消归档流程  
3. 任务验收流程
4. 任务阻塞上报/解决流程
5. 事件重试死信流程
6. API密钥完整生命周期流程
7. 统计查询流程

### 阶段三：前端功能测试（第二天 ~ 第三天）
验证前端页面交互：
1. 表单验证
2. 权限UI展示控制
3. 错误提示正确性
4. 页面跳转流程
5. 响应式布局

### 阶段四：专项测试（第四天）
1. 兼容性测试（多浏览器、多尺寸）
2. 安全测试（越权、XSS）
3. 性能测试（响应时间验证）

### 阶段五：回归测试（第五天 ~ 第六天）
1. 缺陷修复验证
2. 迭代1核心功能冒烟回归
