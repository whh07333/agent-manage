## 三、系统架构设计
### 技术架构
采用标准MVC+Services分层架构：
```
src/
├── models/          # 数据模型
├── controllers/     # 请求处理控制器
├── services/        # 业务逻辑层
├── routes/          # 路由配置
├── middleware/      # 中间件
├── config/          # 配置文件
├── utils/           # 工具类
└── server.ts        # 服务器入口
```

### 现有数据模型
1. **Project.ts**: 项目模型（ID/名称/描述/负责人/优先级/状态/截止日期/归档状态）
2. **Task.ts**: 任务模型（ID/项目ID/名称/描述/负责人/状态/优先级/截止日期）
3. **User.ts**: 用户模型（基础用户信息）

### 待补充数据模型
1. **Subscription.ts**: 事件订阅模型
2. **AuditLog.ts**: 操作审计日志模型
3. **Deliverable.ts**: 交付物管理模型
4. **Role.ts**: 角色权限模型
5. **UserRole.ts**: 用户角色关联模型
6. **ProjectAgent.ts**: 项目成员管理模型

### Git操作规范
1. **分支命名规则**: 
   - 功能开发: feature/功能名称
   - Bug修复: fix/问题描述
   - 重构: refactor/重构范围
   - 文档更新: docs/文档类型
2. **提交信息规范**: `<类型>(<范围>): <简短描述>`
3. **PR流程**: 开发完成→发起PR→审核通过→合并删除分支
4. **忽略文件**: node_modules/、dist/、.env、.vscode/等

## 六、编码规范
1. 代码注释率≥30%
2. 单元测试覆盖率≥60%

4. 命名规范: 驼峰式变量/函数、帕斯卡式类/接口
