### DEF-IT2-3-009: tasks创建接口缺少project_id字段验证 ✅ 已关闭
**用例ID**: -  
**验证时间**: 2026-03-27 01:45（后端重启后验证）

**测试步骤**: 
1. 创建包含project_id的任务
2. 创建不包含project_id的任务

**实际结果**: 
- 包含project_id的任务：✅ 返回code=0，任务创建成功
- 不包含project_id的任务：✅ 返回500错误（Sequelize notNull验证）

**预期结果**: 
- 包含project_id的任务：任务创建成功
- 不包含project_id的任务：应该返回400错误，提示project_id为必填

**验证结果**: ✅ 修复成功，project_id字段验证已生效
**严重程度**: 🔴 P0  
**状态**: ❌ 待修复

**影响范围**: 
- US008 任务验收（16个用例）
- US009 任务阻塞上报（15个用例）
- 共31个用例无法测试

**问题描述**: 
`/Users/whh073/.openclaw/project/AgentManage/backend/src/utils/validation.ts` 中的 `validateTaskData` 函数没有验证 `project_id` 字段，导致任务创建时即使请求中包含 `project_id`，后端也会报告 "projectId cannot be null" 错误。

**根本原因**: 
验证函数中缺少 `project_id` 字段校验逻辑

**建议修复**:
在 `validateTaskData` 函数中添加 `project_id` 字段验证：
```typescript
// 验证project_id
const projectId = data.projectId || data.project_id;
if (!projectId || typeof projectId !== 'string') {
  return { valid: false, message: 'project_id is required and must be a string' };
}
```

**重现步骤**: 
```bash
# 即使包含project_id也会失败
curl -X POST "http://localhost:3001/api/tasks" \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "测试任务",
    "project_id": "{project_id}",
    "priority": "high",
    "status": "in_progress"
  }'
```
