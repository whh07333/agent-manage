## 三、模块详细设计

### 3.1 项目管理模块 - US002 修改项目配置

#### 3.1.1 功能需求
作为管理类Agent，我可以通过界面修改项目配置（周期、目标、关联Agent等），以便应对项目变更需求。

#### 3.1.2 前端设计
- **页面路径**: `/projects/:id/config`
- **布局**: 表单布局，使用 Ant Design `Form` 组件
- **表单字段**:
  | 字段 | 类型 | 验证规则 |
  |------|------|----------|
  | 项目名称 | `Input` | 必填，1-100字符 |
  | 项目描述 | `TextArea` | 可选，最多 1000 字符 |
  | 目标开始日期 | `DatePicker` | 必填 |
  | 目标结束日期 | `DatePicker` | 必填，必须晚于开始日期 |
  | 优先级 | `Select` | 必填，选项 P0/P1/P2/P3 |
  | 关联Agent | `Select` 多选 | 可选 |
  | 自定义字段 | `JsonEditor` | 可选，键值对 |

- **权限控制**:
  - 仅项目负责人或系统管理员可编辑
  - **安全要点**: 前端禁用仅为UX体验，后端必须进行权限校验，防止越权修改
  - 归档项目表单禁用，只读展示
  - 未登录用户直接访问URL跳转登录页
  - 无权限用户显示403提示

- **错误处理**:
  - 网络超时：显示"网络超时，请检查网络连接后重试"
  - 5xx错误：显示"服务器错误，请稍后重试"
  - 参数校验失败：字段下方显示具体错误信息
  - 开始日期晚于结束日期：前端+后端双重校验

- **交互流程**:
  1. 进入页面加载现有项目配置
  2. 用户修改表单内容
  3. 点击保存按钮，前端验证
  4. 调用 `PUT /api/projects/{id}` API
  5. 成功提示，返回项目详情页
  6. 失败显示错误信息

- **并发处理**:
  - 如果配置已被他人修改，前端提示"配置已被他人修改，请刷新后再编辑"
  - 用户可选择刷新或放弃修改

#### 3.1.3 组件设计
```tsx
// 页面组件
ProjectConfig/
  index.tsx          # 主页面
  hooks.ts           # 自定义hooks
  types.ts           # 类型定义
```

#### 3.1.4 状态管理
不需要全局状态，使用页面本地状态 (useState) 即可。

### 3.5 可视化监控模块 - US015 移动端响应式适配

#### 3.5.1 功能需求
人类用户可以在PC、平板、手机端访问，获得一致体验。

#### 3.5.2 响应式断点设计
采用 Tailwind CSS 标准断点：

| 断点范围 | 设备类型 | 布局策略 |
|----------|----------|----------|
| `< 640px` | 手机 | 单列布局，侧边栏折叠为抽屉，菜单收起 |
| `640px - 1024px` | 平板 | 双列布局，侧边栏可折叠 |
| `> 1024px` | PC | 完整多列布局，侧边栏常驻 |

#### 3.5.3 具体适配方案

**布局适配**:
- 整体栅格系统使用 Ant Design Grid `Row` + `Col`，按断点设置不同 `span`
- 侧边栏：移动端隐藏，通过汉堡按钮弹出抽屉
- **表格**: 移动端不使用横向滚动，转为卡片列表展示，提升用户体验

**交互适配**:
- 表格分页：移动端减少每页展示数量 (10 → 5)
- 下拉菜单：移动端改为全屏底部弹窗
- 模态框：移动端占满屏幕，留出顶部关闭栏（关闭按钮 ≥ 48x48px）
- 支持触摸滑动切换标签页和卡片
- 所有可点击元素最小尺寸 ≥ 48x48px，符合 WCAG 可访问性标准

**安全区域适配**:
- 使用 `env(safe-area-inset-bottom)` 适配iPhone底部操作条
- 使用 `env(safe-area-inset-top)` 适配刘海屏
- Android开孔屏也使用相同方案，系统自动处理

**性能优化**:
- 移动端减少动画效果，降低CPU占用
- 图片根据屏幕密度加载不同分辨率
- 按需加载组件，减少初始包大小

**图表适配**:
- 监听窗口 `resize` 事件，图表容器变化时自动重绘
- 不同屏幕尺寸自动调整图表大小和字体

#### 3.5.4 适配范围
所有现有页面 + 新增页面都需要适配：
- ✅ Dashboard 首页
- ✅ 项目列表页
- ✅ 项目详情页
- ✅ 任务列表页
- ✅ 任务详情页
- ✅ 新增的8个页面

#### 3.5.5 测试计划
- **手机端**: iPhone SE (320px), iPhone 14 (390px), Samsung Galaxy (412px)
- **平板**: iPad Mini (768px), iPad Air (820px)
- **桌面**: 1024px, 1440px, 1920px
- **浏览器**: iOS Safari, Android Chrome, 桌面 Chrome/Firefox/Safari

### 3.6 数据统计模块 - US016 多维度统计分析

#### 3.6.1 功能需求
人类用户可以查看统计分析数据（Agent负载、项目效率、异常统计等）。

#### 3.6.2 前端设计
- **页面路径**: `/projects/:id/statistics`
- **布局**: 卡片式概览 + 图表展示 + 数据表格（移动端转为卡片列表）
- **组件设计**:
  - `StatsCard` - 统计卡片，展示数字标题和趋势（遵循设计系统色值）
  - `LineChart` - 基于 Chart.js 封装折线图（配色遵循设计系统）
  - `BarChart` - 基于 Chart.js 封装柱状图（配色遵循设计系统）
  - `WorkloadTable` - Agent负载表格（移动端卡片化）

- **API调用**:
  - 项目概览：`GET /api/statistics/projects/{id}/overview`
  - Agent负载：`GET /api/statistics/agents/{id}/workload`
  - 团队效率：`GET /api/statistics/efficiency`

- **缓存策略**:
  - 统计数据缓存 5 分钟（和后端缓存策略对齐）
  - 用户手动点击刷新按钮可强制刷新

- **权限控制**:
  - 项目级统计：项目成员可访问（后端校验）
  - Agent级统计：Agent本人或管理员可访问（后端校验）
  - 团队效率：仅管理员可访问（后端校验）

- **可测试性**:
  - 图表组件添加 `data-testid` 属性，便于自动化测试定位

- **XSS防护**:
  - 自定义字段和统计描述输出时做HTML转义，防止存储型XSS攻击

#### 3.6.3 可视化展示
- 任务趋势：折线图展示每日创建/完成数量
- Agent负载：柱状图展示各Agent已完成/进行中/阻塞任务
- 进度卡：展示完成率、逾期率等关键指标

### 3.7 审计合规模块 - US018 死信队列管理

#### 3.7.1 功能需求
系统管理员可以管理事件推送失败的死信队列，手动重发失败事件。

#### 3.7.2 前端设计
- **页面路径**: `/admin/subscriptions/dead-letters`
- **布局**: 分页列表 + 操作栏（移动端卡片列表）
- **空状态**: 无数据时显示空状态提示
- **支持自定义每页条数**: 默认20条，可选10/20/50/100
- **列表字段**:
  | 字段 | 展示 |
  |------|------|
  | 事件类型 | 标签展示 |
  | 订阅ID | 链接跳转 |
  | 重试次数 | 数字展示 |
  | 最后错误 | 文本截断，悬浮查看完整 |
  | 创建时间 | 相对时间展示 |

- **操作**:
  - 单条重发：点击重发按钮，调用 `POST /api/subscriptions/dead-letters/{id}/retry`
  - 批量重发：勾选多条，点击"全部重发"，调用 `POST /api/subscriptions/dead-letters/retry-all`
  - 删除：点击删除按钮，调用 `DELETE /api/subscriptions/dead-letters/{id}`
  - 所有操作按钮尺寸 ≥ 48x48px

- **错误处理**:
  - 批量操作部分成功部分失败时，显示成功条数和失败条数，失败的显示具体原因

- **分页**: 默认每页 20 条，支持切换页码

- **并发处理**:
  - 操作完成后自动刷新列表，避免状态不一致

- **权限**: 仅系统管理员可访问（后端必须校验）

### 3.8 审计合规模块 - US019 API密钥全生命周期管理

#### 3.8.1 功能需求
系统管理员可以管理Agent的API密钥（颁发、轮换、吊销）。

#### 3.8.2 前端设计
- **页面路径**: `/admin/agents/:id/api-keys`
- **布局**: 密钥列表 + 新建/轮换弹窗
- **弹窗关闭按钮 ≥ 48x48px**
- **列表字段**:
  | 字段 | 展示 |
  |------|------|
  | 创建时间 | 日期 |
  | 过期时间 | 日期，过期标红 |
  | 状态 | 标签：激活(绿色)/过期(黄色)/吊销(红色)，清晰视觉区分 |
  | 操作 | 按钮：吊销/查看 |

- **状态视觉区分**:
  - ✅ 激活：绿色标签
  - ⚠️ 过期：黄色警告标签
  - ❌ 吊销：红色禁用灰色标签

- **操作**:
  - 新建密钥：点击"新建密钥"，设置过期时间（默认90天），调用后端API创建
  - 轮换密钥：新建密钥后，旧密钥自动进入24小时过渡期
  - 吊销密钥：点击吊销，确认弹窗，吊销后立即失效

- **安全设计**:
  - 创建成功后**只显示一次**完整密钥，提示用户立即保存
  - 创建弹窗增加醒目标题提示："请立即保存密钥，关闭后无法再次查看完整密钥"
  - 提供"显示密钥"按钮（用户主动点击才显示）
  - 列表中只显示密钥前缀（前4位），不展示完整密钥
  - 前端不会存储完整密钥到 localStorage/sessionStorage
  - 吊销后立即从列表中标记为已吊销，状态实时更新

- **权限**: 仅系统管理员可访问（后端必须校验，前端路由守卫双重保护）

  - 请求区域：1小时/24小时请求量、QPS、错误率、平均响应时间
  - 事件区域：总事件数、成功率、失败数、待重试、死信
  - 系统区域：CPU使用率、内存使用率、负载
  - 数据库：活跃连接、平均查询时间、慢查询数

- **刷新策略**:
  - 初始加载自动获取数据
  - 每 30 秒自动刷新
  - **页面隐藏时停止刷新**（tab切换离开），页面显示时恢复刷新
  - 多个标签页同时打开时，只有活跃标签页刷新，节省资源
  - 支持手动点击刷新按钮立即刷新
  - 数据延迟要求 ≤ 1 秒，满足需求

- **网络断开处理**:
  - 连接断开时顶部显示提示条
  - 自动重试连接，重连成功后提示消失
  - 用户可手动点击重试

- **告警提示**:
  - 错误率超过阈值显示红色警告
  - CPU/内存使用率过高显示黄色警告

- **权限**: 仅系统管理员可访问（后端必须校验）

## 五、数据库与状态管理

### 5.1 状态管理设计
| 模块 | 状态管理方式 | 原因 |
|------|-------------|------|
| 项目配置修改 | 本地组件state | 表单提交后跳转，不需要全局状态 |
| 项目归档 | 调用后刷新页面 | 状态在后端，前端不需要持久化 |
| 任务验收/阻塞 | 调用后刷新页面 | 状态在后端 |
| 死信队列管理 | 本地组件state + 分页 | 列表查询，不需要全局状态 |
| API密钥管理 | 本地组件state | 管理页面，低频操作 |
| 统计分析 | 本地组件state + React Query缓存 | 缓存减少重复请求 |
| 实时监控 | 定时器刷新 | 数据需要定期更新 |

### 5.2 缓存策略
- 统计数据：使用 React Query 缓存，缓存时间 5 分钟（和后端对齐）
- 页面数据：合理设置缓存，减少不必要请求
- 实时监控：不缓存，每30秒刷新

## 七、可访问性设计

遵循 WCAG 2.1 AA 标准：
1. **键盘可操作**: 所有交互元素可通过Tab访问，Enter触发
2. **颜色对比度**: 文本对比度 ≥ 4.5:1
3. **ARIA标签**: 所有图标按钮添加aria-label
4. **语义化HTML**: 使用正确的HTML标签（button, a, nav, main 等）
5. **屏幕阅读器**: 重要状态变化使用aria-live通知
6. **点击尺寸**: 所有可点击元素最小尺寸 ≥ 48x48px，避免误触

## 八、性能优化方案

| 优化点 | 方案 | 预期收益 |
|--------|------|----------|
| 路由懒加载 | 使用 React.lazy + Suspense 懒加载各页面 | 减小初始包体积，加快首屏加载 |
| 图表按需加载 | Chart.js 按需导入，只加载用到的组件 | 减小包体积 |
| 图片懒加载 | 使用原生 loading="lazy" | 减少初始请求数 |
| 统计缓存 | React Query 缓存 5 分钟 | 减少重复请求，提升响应速度 |
| 自动刷新控制 | 实时监控 30 秒刷新一次，隐藏页面停止刷新 | 避免不必要的网络请求，节省带宽 |
| 移动端减量 | 移动端减少数据展示条数，表格转卡片 | 加快移动端渲染，减少流量 |
| 滑动性能 | 大数据列表滑动帧率要求 ≥ 50fps | 保证流畅交互 |
