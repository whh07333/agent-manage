## DEF-018: Subscription 模型必填字段缺少 allowNull: false 配置

**问题描述**:
- 模型中所有必填字段 `agentId`, `agentType`, `targetId`, `eventType`, `eventTypes`, `callbackUrl` 都只在 TypeScript 层面做了非null断言 (`!`)，但是 Sequelize `init` 配置中没有添加 `allowNull: false`
- 导致运行时 Sequelize 认为这些字段允许null，但是实际上业务上必填，不添加配置会导致允许null不符合业务要求

**修复方式**:
在 `init` 配置中给每个必填字段添加 `allowNull: false`

## DEF-019: Subscription 模型错误多定义了 eventType 字段

**问题描述**:
- 业务需要订阅多个事件类型 → 模型字段应该是 `eventTypes: string[]`
- 模型错误定义了 `eventType: string` 多余字段 → 创建时提示 `eventType cannot be null`
- 删除这个多余字段就能修复

## DEF-020: Subscription 模型 projectId 缺少 allowNull: false 配置

**问题描述**:
- 模型 `projectId` 定义了非null断言，但是 init 配置中缺少 `allowNull: false`
- Sequelize 运行时认为该字段允许null，不符合业务需求，创建时必须传入projectId

**修复方式**:
```typescript
projectId: {
  type: DataTypes.UUID,
  allowNull: false,  // 添加这行
},
