## 缺陷详细描述

### E2E-001: 项目卡片onClick事件未配置 🔴

**缺陷等级**: 🔴 S1（严重）  
**优先级**: 🔴 P0（阻塞性）

#### 问题描述

点击项目卡片后，页面URL没有从 `/projects` 变为 `/projects/:id`，导致无法进入项目详情页。

**影响范围**: 整个任务管理模块的详情页访问

**复现步骤**：
1. 访问项目列表页面 `/projects`
2. 点击第一个项目卡片
3. 等待5秒观察URL变化
4. 实际：URL仍然是 `/projects`，未变为 `/projects/:id`

**实际结果**:
```
[导航] 点击前URL: http://localhost:5173/projects
✓ 点击了第一个项目卡片
❌ TimeoutError: page.waitForURL: Timeout 5000ms exceeded.
```

**期望结果**: URL应变为 `http://localhost:5173/projects/:id`

**根因分析**:
项目卡片的 `onClick` 事件未配置或配置不正确，没有调用路由导航函数。

**修复建议**:
1. 检查前端项目卡片的 `onClick` 事件配置
2. 确保调用了路由导航函数 `navigate(`/projects/${projectId}\`)`
3. 验证项目ID正确传递
4. 参考代码示例：
```tsx
const { navigate } = useNavigate();

{projects.map(project => (
  <div 
    className="project-item"
    onClick={() => navigate(`/projects/${project.id}`)}  // 确保这里配置了onClick
  >
    {/* 项目内容 */}
  </div>
))}
```

**验证步骤**:
1. 前端开发修复后，手动访问项目列表
2. 点击项目卡片
3. 观察URL是否正确变为 `/projects/:id`
4. 确认能进入项目详情页
5. 验证任务列表正常显示

**测试用例影响**:
- E2E-P002: 点击项目卡片能进入详情页 - 被阻塞
- 所有依赖项目详情页的测试用例 - 被阻塞

### E2E-002: Ant Design Modal overlay拦截pointer events 🔴

**缺陷等级**: 🔴 S1（严重）  
**优先级**: 🔴 P0（阻塞性）

#### 问题描述

创建项目的Modal框打开后，点击Modal中的"保存"按钮时，Ant Design的 `div class="ant-modal-wrap"` 拦截了pointer events，导致按钮点击无法传递到实际按钮元素，操作超时失败。

**影响范围**: 所有Modal表单操作（创建、编辑等）

**复现步骤**:
1. 访问项目列表页面 `/projects`
2. 点击"创建项目"按钮
3. 等待Modal框打开
4. 填写项目信息
5. 点击"保存"按钮
6. 观察：Playwright提示 overlay拦截

**实际结果**:
```
<div class="ant-modal-wrap">…</div> from <div>…</div> subtree intercepts pointer events
TimeoutError: locator.click: Timeout 30000ms exceeded.
```

**期望结果**: 保存按钮点击成功，Modal关闭，项目创建成功

**根因分析**:
Ant Design Modal组件的 overlay 层（`ant-modal-wrap`）默认设置 `pointer-events: none`，会拦截所有点击事件。这是Ant Design的设计特性，但在自动化测试中需要特殊处理。

**修复建议**:

**方案A（推荐）**: 前端添加 `data-testid` 属性
```tsx
<Modal
  data-testid="create-project-modal"
  ...
>
  <Button 
    data-testid="create-project-save-button"
    type="primary"
    onClick={handleSave}
  >
    保存
  </Button>
</Modal>
```

**方案B**: 使用 `forceClick()` 绕过事件拦截
```tsx
await page.locator('[data-testid="save-button"]').click({ force: true });
```

**方案C**: 关闭Modal overlay
```tsx
<Modal mask={false} maskClosable={false}>
```

**验证步骤**:
1. 前端添加 `data-testid` 属性到关键按钮
2. 重新运行E2E测试
3. 验证创建项目Modal能正常保存
4. 验证项目创建成功
5. 验证项目出现在列表中

**测试用例影响**:
- E2E-P003: 创建项目Modal可用 - 被阻塞
- 所有Modal相关测试用例 - 被阻塞
