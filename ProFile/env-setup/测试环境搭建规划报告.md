## 1. 现状分析

### 1.1 当前环境状况

| 组件 | 现状 | 占用端口 | 使用者 |
|------|------|---------|--------|
| PostgreSQL | 已容器化运行 | 5433 | 开发环境 |
| 后端服务 | 宿主机运行 | 3000 | 开发环境 |
| 前端服务 | 宿主机运行 | 5173 | 开发环境 |

### 1.2 存在的问题

1. **数据库不隔离**: 目前开发直接使用容器中的数据库，测试若共用会导致测试数据污染开发数据
2. **服务端口不隔离**: 若测试共用端口，开发调试会影响测试执行
3. **数据不一致**: 测试环境需要干净的初始数据，不能混入开发调试数据

### 1.3 设计原则

遵循以下原则进行测试环境设计：
1. **完整隔离原则**: 开发和测试从数据库到应用层完全隔离
2. **互不影响原则**: 测试环境启停、数据重置不影响开发环境正常工作
3. **可重复性原则**: 测试环境可以快速重建、重置数据
4. **端口唯一原则**: 不同环境使用不同端口，避免端口冲突

## 2. 测试环境架构设计

### 2.1 环境隔离方案

采用**三层全隔离架构**，确保开发测试完全独立：

| 层级 | 开发环境 | 测试环境 | 隔离方式 |
|------|---------|---------|---------|
| 数据库 | agentmanage-postgres (容器) | agentmanage-test-postgres (新容器) | 不同容器/不同端口/不同数据库 |
| 后端服务 | 端口 3000 | 端口 3001 | 端口隔离 |
| 前端服务 | 端口 5173 | 端口 5174 | 端口隔离 |

### 2.2 最终配置对照表

| 配置项 | 开发环境 | 测试环境 |
|--------|---------|---------|
| **容器名称** | `agentmanage-postgres` | `agentmanage-test-postgres` |
| **数据库端口** | 5433 → 5432 | 5434 → 5432 |
| **数据库名** | `agentmanage` | `agentmanage_test` |
| **用户名** | `postgres` | `postgres` |
| **密码** | `postgres` | `postgres` |
| **后端端口** | 3000 | 3001 |
| **前端端口** | 5173 | 5174 |
| **API地址** | `http://localhost:3000` | `http://localhost:3001` |
| **前端访问地址** | `http://localhost:5173` | `http://localhost:5174` |
| **环境配置文件** | `.env.development` | `.env.test` |
| **数据卷** | `postgres_data` | `postgres_test_data` |

```

│                                   │     │                                   │

│                                   │ │ │ │                                   │
│          ↓                       │ │ │ ↓                                   │
│  Postgres: localhost:5433        │ │ │  Postgres: localhost:5434 (test)    │
│  Database: agentmanage           │ │ │  Database: agentmanage_test         │
└─────────────────────────────────┘ │ │ └─────────────────────────────────┘

    完全网络隔离 (不同端口)                          数据完全隔离 (不同容器)
```

## 4. 配置文件示例

### 4.1 docker-compose.yml 新增配置

```yaml
  postgres-test:
    image: postgres:14-alpine
    container_name: agentmanage-test-postgres
    restart: always
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: agentmanage_test
    ports:
      - "5434:5432"
    volumes:
      - postgres_test_data:/var/lib/postgresql/data
      - ./backend/init-db.sql:/docker-entrypoint-initdb.d/init-db.sql
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 5s
      timeout: 5s

volumes:
```

## 5. 环境启停脚本

为了方便测试人员快速启动/停止测试环境，提供快捷脚本：

### 5.1 scripts/start-test.sh

```bash
#!/bin/bash
echo "Starting test environment..."

# 启动测试数据库
docker-compose up -d postgres-test

# 启动测试后端
cd ../backend
cp .env.test .env
npm run build
npm run start &
echo "Backend started on port 3001"

# 启动测试前端
cd ../frontend
cp .env.test .env.development
npm run dev -- --port 5174 &
echo "Frontend started on port 5174"

echo "Test environment ready:"
echo "Frontend: http://localhost:5174"
echo "Backend: http://localhost:3001"
echo "Database: localhost:5434/agentmanage_test"
```

### 5.2 scripts/stop-test.sh

```bash
#!/bin/bash
echo "Stopping test environment..."

# 停止后端和前端进程
pkill -f "node dist/server.js.*PORT=3001"
pkill -f "vite.*port 5174"

# 测试数据库保持运行，不需要每次停止
echo "Test environment stopped."
```

### 5.3 scripts/reset-test-db.sh

```bash
#!/bin/bash
echo "Resetting test database..."
docker exec -it agentmanage-test-postgres psql -U postgres -c "drop database if exists agentmanage_test;"
docker exec -it agentmanage-test-postgres psql -U postgres -c "create database agentmanage_test;"
cat ../backend/init-db.sql | docker exec -i agentmanage-test-postgres psql -U postgres -d agentmanage_test
echo "Test database reset completed."
```

## 6. 风险评估与应对

| 风险 | 影响等级 | 应对措施 |
|------|---------|---------|
| 端口被占用 | 中 | 启动时检查端口占用，若被占用提示用户更换端口 |
| 镜像拉取失败 | 低 | 使用已有的postgres:14-alpine镜像，本地已存在 |
| 初始化脚本执行失败 | 中 | 脚本执行后验证表数量，失败立即报警 |
| 开发环境被意外影响 | 低 | 使用完全不同的容器和数据卷，理论上不会影响 |
| 磁盘空间不足 | 低 | 两个PG容器总占用不到1GB，空间充足 |

## 7. 验证标准

测试环境搭建完成后，需通过以下验证：

- [ ] 测试数据库容器运行状态为 `Up (healthy)`
- [ ] 测试数据库 `agentmanage_test` 创建成功
- [ ] 所有初始化表创建完成，数量与开发环境一致
- [ ] 测试后端服务在3001端口正常监听
- [ ] 测试前端服务在5174端口正常监听
- [ ] 测试前端能正常访问测试后端API
- [ ] 开发环境所有服务仍正常运行（回归验证）
- [ ] 开发者可以正常继续开发，不受测试环境影响

全部验证通过后，测试环境搭建完成，可以开始测试执行。
